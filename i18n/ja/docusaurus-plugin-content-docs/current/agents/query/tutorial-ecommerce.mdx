このチュートリアルでは、**[Weaviate Query エージェント](./index.md)** を使用してシンプルな e-commerce アシスタント エージェントを構築します。エージェントは複数の Weaviate コレクションにアクセスし、各コレクションの情報を使ってブランドや衣料品に関する複雑な質問に回答できるようになります。

Query エージェントを試せる公開データセットを用意しました。  
HuggingFace で利用できます。

- [**E-Commerce:**](https://huggingface.co/datasets/weaviate/agents/viewer/query-agent-ecommerce) 衣料品、価格、ブランド、レビューなどを掲載したデータセット  
- [**Brands:**](https://huggingface.co/datasets/weaviate/agents/viewer/query-agent-brands) ブランドの親ブランドや子ブランド、平均顧客評価などの情報を掲載したデータセット

## Query エージェントとは

Weaviate Query エージェントは、Weaviate Cloud に保存されたデータを基に自然言語クエリに回答するために設計された、あらかじめ構築済みのエージェント サービスです。ユーザーは自然言語でプロンプト／質問を渡すだけで、Query エージェントが回答までのすべての手順を処理します。

Query エージェントは次のように動作します。

1. **タスクを受け取る** — 質問や instructions の形で渡されます。  
2. **適切な生成モデル**（例: 大規模言語モデル）を使用してタスクを解析し、実行すべき正確なクエリを決定します。  
3. **Weaviate にクエリを送信** — 指定されたベクトライザー統合（ここでは Weaviate Embeddings）を使用して必要に応じてクエリをベクトル化します。  
4. **Weaviate から結果を受け取り**、適切な生成モデルを用いてユーザーのプロンプト／クエリに対する最終的な回答を生成します。

## 前提条件

Weaviate エージェントおよび Weaviate Embedding サービスを利用するには、**[Weaviate Cloud](https://console.weaviate.cloud)** のアカウントが必要です。

## Step 1: Weaviate をセットアップする

ここからは、チュートリアルで使用する Weaviate Cloud インスタンスをセットアップし、Python クライアントから接続します。

### 1.1 Weaviate Cloud クラスターの作成

1. Weaviate Cloud で [無料の Sandbox クラスター](/cloud/manage-clusters/create#sandbox-clusters) を作成します。  
2. クラスターへ接続するために `REST Endpoint` と `Admin` API キーを控えます。（詳細は [クイックスタート](/cloud/manage-clusters/connect.mdx) を参照）

:::tip
本チュートリアルではベクトライザーとして [Weaviate Embeddings](/weaviate/model-providers/weaviate/embeddings.md) サービスを使用するため、外部の埋め込みプロバイダー用キーを追加する必要はありません。Weaviate Embeddings はデフォルトで `Snowflake/snowflake-arctic-embed-l-v2.0` モデルを使用します。<br/><br/>
他のベクトライザーを使用したい場合は、サポートされている [モデル プロバイダー](../../weaviate/model-providers/index.md) の一覧をご覧ください。
:::

## ステップ 4: Query Agent を実行する

エージェントを実行すると、クエリに応じてまず次の決定を行います。

1. どのコレクション（複数の場合もあり）を検索するかを決定します。  
2. **_検索クエリ_** を行うか、どの **_フィルター_** を使うか、**_集計クエリ_** を行うか、またはそれらすべてを組み合わせるかを判断します。  
3. その後、 **`QueryAgentResponse`** として応答を返します。

### 4.5 システムプロンプトの使用 {#use-system-prompt-agent}

[ステップ 3.2](#system-prompt-agent) で作成したカスタムエージェントを、次のシステムプロンプトと共に使用します:

```
"You are a helpful assistant that always generates the final response in the user's language."
"You may have to translate the user query to perform searches. But you must always respond to the user in their own language."
```

例えば、ブランド「Eko & Stitch」に英国で支店または関連会社があるかどうかを、英語ではなく日本語で尋ねてみてください:

## Summary

本ガイドでは、Weaviate のエージェントサービスを使用してエンドツーエンドの EC アシスタントを構築する方法を説明しました。これには、Weaviate Cloud インスタンスのセットアップ、データセットのインポート、自然言語クエリをインテリジェントに処理する Query Agent の設定が含まれます。

Query Agent はクエリを自動的に解釈し、通常の検索・フィルター・集計のいずれを実行するかを判断します。また、複数のコレクションに接続し、クエリに基づいて適切なソースを選択します。さらに、過去の対話を利用してより関連性の高いフォローアップ回答を行い、システムプロンプトを定義することで、ユーザーの希望する言語やスタイルで常に応答させることができます。
