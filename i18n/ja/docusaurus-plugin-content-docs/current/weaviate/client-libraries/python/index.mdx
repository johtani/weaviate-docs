## インストール

Python クライアントライブラリは Python 3.8 以上で開発・テストされています。[PyPI.org](https://pypi.org/project/weaviate-client/) から入手でき、次のコマンドでインストール可能です。

```bash
pip install -U weaviate-client
```

<details>
  <summary>ベータ版をインストールする場合</summary>

```bash
pip install --pre -U "weaviate-client==4.*"`
```

</details>

<details>
  <summary>要件:  Weaviate バージョン互換性 & gRPC</summary>

#### Weaviate バージョン互換性

`v4` Python クライアントは Weaviate `1.23.7` 以降が必要です。一般的に、最新バージョンの Python クライアントと Weaviate データベースを併用することを推奨します。

Weaviate Cloud では、2024 年 1 月 31 日以降に作成された Sandbox が `v4` クライアントと互換性があります。この日以前に作成された Sandbox は `v4` クライアントと互換性がありません。

#### gRPC

`v4` クライアントは内部でリモートプロシージャコール (RPC) を使用します。そのため、Weaviate サーバーに対して gRPC 用のポートを開放しておく必要があります。

<details>
  <summary>docker-compose.yml の例</summary>

Docker で Weaviate を実行している場合、デフォルトポート（ `50051` ）を次のように `docker-compose.yml` でマッピングできます。

```yaml
ports:
  - 8080:8080
  - 50051:50051
```

</details>

</details>

#### Weaviate エージェント

[Weaviate エージェント](../../../agents/index.md) を利用するために、オプションの agents エクストラ付きで Weaviate クライアントライブラリをインストールできます。次のコマンドを実行してください。

```bash
pip install -U weaviate-client[agents]
```

## 非同期での利用

Python クライアントライブラリはデフォルトで `WeaviateClient` クラスを通じた同期 API を提供します（本ページで説明）。`weaviate-client` `v4.7.0` 以降では、`WeaviateAsyncClient` クラスによる非同期 API も利用可能です。詳細は [非同期クライアント API ページ](./async.md) を参照してください。

#### ベクトライザー API の変更点 `v4.16.0`

<VectorsAutoSchemaError />

Weaviate Python クライアント `v4.16.0` 以降、コレクション作成時のベクトライザー設定 API に複数の変更があります:
- `.vectorizer_config` は `.vector_config` に置き換えられました
- `Configure.NamedVectors` は `Configure.Vectors` と `Configure.MultiVectors` に置き換えられました
- `Configure.NamedVectors.none` と `Configure.Vectorizer.none` は `Configure.Vectors.self_provided` と `Configure.MultiVectors.self_provided` に置き換えられました

Python `v3` クライアントから `v4` クライアントへ移行する場合は、この [専用ガイド](./v3_v4_migration.md) を参照してください。

#### ベータ版リリース

<details>

#### `v4.4b9` の変更点

##### `weaviate.connect_to_x` メソッド

`timeout` 引数は `additional_config` 引数の一部になりました。クラス `weaviate.config.AdditionalConfig` を入力として受け取ります。

##### クエリ

`query` 名前空間のメソッドに対するすべてのオプション引数は、キーワード引数としてのみ指定する必要があります。

クエリ引数の型を強制するランタイム解析ロジックが追加されました。

##### バッチ処理

内部的に異なるバッチング方式を用いる 3 つのアルゴリズムが導入されました:

- `client.batch.dynamic()`
- `client.batch.fixed_size()`
- `client.batch.rate_limit()`

`client.batch.dynamic() as batch` は、以前の `client.batch as batch` のドロップイン置換です。後者は非推奨となり、正式リリース時に削除されます。

```python
with client.batch.dynamic() as batch:
  ...
```

は次と同等です:

```python
with client.batch as batch:
  ...
```

`client.batch.fixed_size() as batch` は、固定サイズのみを使用するバッチング アルゴリズムを設定する方法です。

```python
with client.batch.dynamic() as batch:
  ...
```

は次と同等です:

```python
client.batch.configure_fixed_size()
with client.batch as batch:
  ...
```

`client.batch.rate_limit() as batch` は、サードパーティのベクトル化 API のレート制限に抵触しないよう支援する新しい方法です。`rate_limit()` メソッドで `request_per_minute` を指定することで、サードパーティ API が処理可能な速度で Weaviate へオブジェクトを送信するようバッチング アルゴリズムを制御できます。

これらのメソッドは完全にローカライズされたコンテキストマネージャを返します。つまり、一度のバッチでの `failed_objects` や `failed_references` は、後続の呼び出しには含まれません。

最後に、バッチ送信を担当するバックグラウンド スレッドで例外が発生した場合、静かに失敗するのではなくメイン スレッドで再スローされます。

##### フィルター

`Filter.by_property` の引数 `prop` は `name` にリネームされました。

参照カウントは `Filter([ref])` ではなく `Filter.by_ref_count(ref)` を使用して実現します。

#### `v4.4b8` の変更点

##### 参照フィルター

参照フィルターの構文が簡素化されました。新しい構文は次のとおりです:

```python
Filter.by_ref("ref").by_property("target_property")
```

#### `v4.4b7` の変更点

##### ライブラリの import

`weaviate` から直接 import する方法は非推奨です。代わりに `import weaviate.classes as wvc` を使用してください。

##### クライアント接続のクローズ

v4.4b7 からはクライアント接続を明示的にクローズする必要があります。接続をクローズする方法は 2 つあります。

`client.close()` を使って明示的にクローズする。

```python
import weaviate
client = weaviate.connect_to_local()

print(client.is_ready())

client.close()
```

コンテキストマネージャを使用して自動でクローズさせる。

```python
import weaviate

with weaviate.connect_to_local() as client:
     print(client.is_ready())

# Python closes the client when you leave the 'with' block
```

##### バッチ処理

v4.4b7 クライアントでは `client.batch` に変更があります。

- `client.batch` はコンテキストマネージャが必須です。
- マニュアル モードは削除され、`.create_objects` でバッチを送信することはできません。
- バッチサイズと同時リクエスト数は動的に割り当てられます。値を指定するには `batch.configure_fixed_size` を使用してください。
- `add_reference` メソッドが更新されました。
- `to_object_collection` メソッドは削除されました。

更新後の `client.batch` パラメーター

| 旧値                                    | v4.4b7 での値                           |
| :-------------------------------------- | :-------------------------------------- |
| from_object_uuid: UUID                  | from_uuid: UUID                         |
| from_object_collection: str             | from_collection: str                    |
| from_property_name: str                 | from_property: str                      |
| to_object_uuid: UUID                    | to: Union[WeaviateReference, List[UUID]]|
| to_object_collection: Optional[str] = None |                                         |
| tenant: Optional[str] = None            | tenant: Optional[str] = None            |

##### フィルター構文

v4.4b7 でフィルター構文が更新されました。

**NOTE**: [参照フィルター構文](#reference-filters) は 4.4b8 で簡素化されています。

| 旧構文                                                 | v4.4b7 の新構文                                                             |
| :---------------------------------------------------- | :-------------------------------------------------------------------------- |
| Filter(path=property)                                 | Filter.by_property(property)                                                |
| Filter(path=["ref", "target_class", "target_property"])| Filter.by_ref().link_on("ref").by_property("target_property")               |
| FilterMetadata.ByXX                                   | Filter.by_id()<br/> Filter.by_creation_time()<br/> Filter.by_update_time()  |

v4.4b7 以前のフィルター構文は非推奨です。新しい v4.4b7 の構文は次のようになります。

```python
import weaviate
import datetime
import weaviate.classes as wvc

client = weaviate.connect_to_local()

jeopardy = client.collections.get("JeopardyQuestion")
response = jeopardy.query.fetch_objects(
    filters=wvc.query.Filter.by_property("round").equal("Double Jeopardy!") &
            wvc.query.Filter.by_creation_time().greater_or_equal(datetime.datetime(2005, 1, 1)) |
            wvc.query.Filter.by_creation_time().greater_or_equal(datetime.datetime(2000, 12, 31)),
            limit=3
    )


client.close()
```


##### `reference_add_many` の更新

`reference_add_many` の構文が更新され、`DataReferenceOneToMany` は `DataReference` になりました。

```python
collection.data.reference_add_many(
    [
        DataReference(
            from_property="ref",
            from_uuid=uuid_from,
            to_uuid=*one or a list of UUIDs*,
        )
    ]
)
```

##### 参照

マルチターゲット参照が更新されました。新しい関数は次のとおりです:

- `ReferenceProperty.MultiTarget`
- `DataReference.MultiTarget`
- `QueryReference.MultiTarget`

マルチターゲット参照には `ReferenceToMulti` を使用してください。

#### 旧クライアントの変更点

##### 参照

- 参照は、コレクション作成・オブジェクト挿入・クエリ実行時に `references` パラメーターを通じて追加されるようになりました。
- `FromReference` クラスは `QueryReference` と呼ばれるようになりました。

##### クラス／パラメーターの再編成

- `weaviate.classes` サブモジュールがさらに分割されました:
  - `weaviate.classes.config`
  - `weaviate.classes.data`
  - `weaviate.classes.query`
  - `weaviate.classes.generic`
- `wvc.config.Configure` および `wvc.config.Reconfigure` の `vector_index_config` パラメーター向けファクトリー関数が変更されました。例:
  ```python
  client.collections.create(
      name="MyCollection",
      # highlight-start
      vector_index_config=wvc.config.Configure.VectorIndex.hnsw(
          distance_metric=wvc.config.VectorDistances.COSINE,
          vector_cache_max_objects=1000000,
          quantizer=wvc.config.Configure.VectorIndex.Quantizer.pq()
      ),
      # highlight-end
  )
  ```
  - `vector_index_type` パラメーターは削除されました。
- `Property` コンストラクターの `vectorize_class_name` パラメーターは `vectorize_collection_name` になりました。
- `[collection].data.update()` / `.replace()` の *args の順序が変更され、更新時にプロパティを省略できるようになりました。
- `[collection].data.reference_add` / `.reference_delete` / `.reference_replace` のキーワード `ref` は `to` に改名されました。
- `collections.create()` / `get()` のジェネリクスを指定するキーワード `data_model` は `data_model_properties` に改名されました。
- `[object].metadata.uuid` は `[object].uuid` になりました。
- `[object].metadata.creation_time_unix` は `[object].metadata.creation_time` になりました。
- `[object].metadata.last_update_time_unix` は `[object].metadata.last_update` になりました。
- `quantitizer` は `quantizer` に改名されました。
- 返却データにベクトルを含めるには `include_vector` パラメーターを使用してください。

##### データ型

- 作成日時および最終更新日時のメタデータは `datetime` オブジェクトを返すようになり、`MetadataQuery` ではそれぞれ `creation_time` と `last_update_time` に改名されました。
  - `metadata.creation_time.timestamp() * 1000` は以前と同じ値を返します。
- `query.fetch_object_by_id()` は内部で REST ではなく gRPC を使用し、他のクエリと同じフォーマットでオブジェクトを返します。
- `UUID` および `DATE` プロパティは型付きオブジェクトとして返されます。
